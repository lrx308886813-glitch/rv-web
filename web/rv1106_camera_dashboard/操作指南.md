# RV1106 Camera Dashboard 操作说明

这个目录是一套本机网页监控台，用 Windows 电脑把 RV1106 的 RTSP 流转换成浏览器可看的 WebRTC/HLS 页面，并支持录制、回放、画质预设和画幅显示调整。

## 目录说明

```text
E:\code\rvtest\web\rv1106_camera_dashboard
```

核心文件：

```text
index.html              网页界面
camera_site_server.py   本机后端，提供录制、目录选择、画质切换 API
mediamtx.exe            RTSP 转 WebRTC/HLS 的媒体网关
mediamtx.yml            MediaMTX 配置
start_camera_site.cmd   启动脚本
stop_camera_site.cmd    停止脚本
recordings              录像保存目录
操作指南.md             当前说明文档
```

临时文件：

```text
*.log
__pycache__
rkipc.generated.ini
```

这些是运行时生成文件，可以清理；录像目录 `recordings` 不建议随便删除。

## 前置条件

正常运行需要同时满足：

- RV1106 和 Windows 电脑在同一个局域网。
- 当前 RV1106 LAN IP：`192.168.31.18`。
- RV1106 RTSP 端口：`554`。
- RV1106 ADB 地址：`192.168.31.18:5555`。
- Windows 上 ADB 路径：`E:\dev\ADB\platform-tools\adb.exe`。
- Windows 上不能占用这些端口：`8080`、`8888`、`8889`、`9996`、`9997`。
- 板卡视频编码必须是 H.264，浏览器播放 H.265 通常会失败。

## 一键启动

双击运行：

```powershell

```

启动后打开：

```text

```

本机会启动这些服务：

```text

http://localhost:8888/sub          子码流 HLS
http://localhost:9996              历史录像回放服务
http://127.0.0.1:9997              MediaMTX 本机 API
```

```powershell
E:\code\rvtest\web\rv1106_camera_dashboard\stop_camera_site.cmd
```

## 页面使用

左侧 `实况水晶`：

- `主码流`：`rtsp://192.168.31.18/live/0`
- `子码流`：`rtsp://192.168.31.18/live/1`
- `WebRTC`：低延迟，优先使用
- `HLS`：兼容性更稳，但延迟更高

左侧 `画质调校`：

- `超清`：主码流 2304x1296
- `高清`：主码流 1920x1080
- `均衡`：主码流 1280x720
- `流畅`：子码流 704x576

清晰度按钮会通过 ADB 修改板卡配置，并重启 `rkipc`。切换时画面短暂中断是正常的。录制中不能切换清晰度，需要先停止录制。

左侧 `画幅比例`：

- `自适应`：跟随当前码流比例
- `16:9`：强制宽屏显示
- `4:3`：强制标准比例
- `铺满`：填满播放器区域

画幅比例只影响网页显示，不修改板卡真实编码。

## 录制和回放

默认录像目录：

```text
E:\code\rvtest\web\rv1106_camera_dashboard\recordings
```

操作流程：

1. 在 `影像卷轴` 里选择或输入保存目录。
2. 点 `开始录制`。
3. 录制完成后点 `停止录制`。
4. 在 `历史回放` 里刷新并选择录像片段。

录像会按码流分目录：

```text
recordings\main\...
recordings\sub\...
```

## 常用检查命令

检查本机网页服务：

```powershell
Invoke-WebRequest -UseBasicParsing http://127.0.0.1:8080/api/status
```

检查本机端口：

```powershell
Get-NetTCPConnection -LocalPort 8080,8888,8889,9996 -State Listen
```

检查板卡 RTSP 端口：

```powershell
Test-NetConnection 192.168.31.18 -Port 554
```

连接板卡 ADB：

```powershell
E:\dev\ADB\platform-tools\adb.exe connect 192.168.31.18:5555
```

查看板卡 IP：

```powershell
E:\dev\ADB\platform-tools\adb.exe -s 192.168.31.18:5555 shell ifconfig
```

查看 `rkipc` 是否运行：

```powershell
E:\dev\ADB\platform-tools\adb.exe -s 192.168.31.18:5555 shell "ps | grep rkipc"
```

查看板卡是否监听 RTSP：

```powershell
E:\dev\ADB\platform-tools\adb.exe -s 192.168.31.18:5555 shell "netstat -lntp 2>/dev/null | grep ':554' || netstat -an | grep ':554'"
```

检查编码格式：

```powershell
E:\dev\ADB\platform-tools\adb.exe -s 192.168.31.18:5555 shell "grep -n output_data_type /userdata/rkipc.ini"
```

## 没画面时怎么排查

先分清是哪一段坏了：

```text
RV1106 摄像头/rkipc -> RTSP 554 -> MediaMTX -> 浏览器网页
```

### 1. 网页打不开

先启动本机服务：

```powershell
E:\code\rvtest\web\rv1106_camera_dashboard\start_camera_site.cmd
```

再检查：

```powershell
Invoke-WebRequest -UseBasicParsing http://127.0.0.1:8080/api/status
```

如果 8080 没响应，看日志：

```powershell
Get-Content E:\code\rvtest\web\rv1106_camera_dashboard\site.err.log -Tail 80
```

### 2. 网页能打开，但播放器报错

先看 MediaMTX 是否启动：

```powershell
Get-NetTCPConnection -LocalPort 8889,8888,9996 -State Listen
```

再看 MediaMTX 日志：

```powershell
Get-Content E:\code\rvtest\web\rv1106_camera_dashboard\mediamtx.out.log -Tail 120
```

如果日志里出现 `H265`，说明板卡编码不是 H.264。打开网页点一次 `超清`，或者用下面命令改：

```powershell
E:\dev\ADB\platform-tools\adb.exe -s 192.168.31.18:5555 shell "sed -i 's/output_data_type[[:space:]]*=[[:space:]]*H.265/output_data_type = H.264/g' /userdata/rkipc.ini"
```

### 3. 554 端口不通

说明板卡 RTSP 没起来。先用 ADB 看 `rkipc`：

```powershell
E:\dev\ADB\platform-tools\adb.exe connect 192.168.31.18:5555
E:\dev\ADB\platform-tools\adb.exe -s 192.168.31.18:5555 shell "ps | grep rkipc"
```

按原厂脚本重启板卡应用：

```powershell
E:\dev\ADB\platform-tools\adb.exe -s 192.168.31.18:5555 shell "/etc/init.d/S21appinit stop; sleep 1; /etc/init.d/S21appinit start"
```

如果原厂脚本启动后还是没有 554，使用带环境变量的启动方式：

```powershell
E:\dev\ADB\platform-tools\adb.exe -s 192.168.31.18:5555 shell "kill -9 $(pidof rkipc) 2>/dev/null; sleep 1; source /etc/profile.d/RkEnv.sh; cd /oem; start-stop-daemon -S -b -x /oem/usr/bin/rkipc -- -a /oem/usr/share/iqfiles"
```

如果仍然不行，可以临时前台启动看错误：

```powershell
E:\dev\ADB\platform-tools\adb.exe -s 192.168.31.18:5555 shell "source /etc/profile.d/RkEnv.sh; cd /oem; /oem/usr/bin/rkipc -a /oem/usr/share/iqfiles"
```

这个命令如果一直不退出，说明 `rkipc` 正在前台运行；如果很快退出，终端里会显示原因。

### 4. 554 能连接但 RTSP 不回包

这种情况一般是 `rkipc` 的 RTSP 线程卡住。处理顺序：

1. 停止本机网站，避免 MediaMTX 一直重连。
2. 重启板卡 `rkipc`。
3. 确认 554 端口正常。
4. 再启动本机网站。

命令：

```powershell
E:\code\rvtest\web\rv1106_camera_dashboard\stop_camera_site.cmd
E:\dev\ADB\platform-tools\adb.exe -s 192.168.31.18:5555 shell "kill -9 $(pidof rkipc) 2>/dev/null; sleep 1; source /etc/profile.d/RkEnv.sh; cd /oem; start-stop-daemon -S -b -x /oem/usr/bin/rkipc -- -a /oem/usr/share/iqfiles"
E:\code\rvtest\web\rv1106_camera_dashboard\start_camera_site.cmd
```

## 板卡配置说明

板卡当前关键配置：

```text
/userdata/rkipc.ini
/tmp/rkipc-factory-config.ini
/oem/usr/share/rkipc-300w.ini
```

注意：原厂启动脚本可能会把 `/tmp/rkipc-factory-config.ini` 复制到 `/userdata/rkipc.ini`。如果只改 `/userdata/rkipc.ini`，板卡重启后可能又恢复成 H.265。

本项目的清晰度接口会尽量同时修改：

```text
/userdata/rkipc.ini
/tmp/rkipc-factory-config.ini 指向的默认配置
```

## Tailscale 远程访问

Windows 电脑登录 Tailscale 后，远程设备打开：

```text
http://<这台Windows电脑的Tailscale-IP>:8080/index.html
```

需要能访问这些端口：

```text
8080/tcp   网页
8889/tcp   WebRTC 页面入口
8189/udp   WebRTC ICE
8888/tcp   HLS 备用播放
9996/tcp   历史录像回放
```

远程网络下 WebRTC 不稳定时，切到页面里的 `HLS`。

目前本机网页项目已经具备：
